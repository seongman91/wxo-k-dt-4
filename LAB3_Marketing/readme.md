# 3교시 핸즈온: 마케팅 에이전트 만들기

## 1. 실습 개요

이번 실습에서는 IBM watsonx Orchestrate에서 **마케팅 업무 지원 Agent**를 직접 생성합니다.

이 Agent는 사내 직원들의 마케팅 관련 질문에 답변하고, 온라인 홍보, 키워드 활용, 블로그 마케팅, 카카오스토리, 페이스북, 포털사이트 등록 방법 등을 안내하는 역할을 합니다.

이번 실습의 핵심은 단순히 Agent를 만드는 것이 아니라, Agent 품질을 결정하는 세 가지 요소를 이해하는 것입니다.

* **Description**: Agent의 역할과 업무 범위 정의
* **Knowledge**: Agent가 참고할 지식의 범위와 설명 정의
* **Behavior**: Agent가 어떻게 답변해야 하는지 행동 방식 정의

---

## 2. 실습 목표

실습이 끝나면 다음을 수행할 수 있습니다.

* Marketing Agent를 직접 생성할 수 있다.
* Agent 이름과 설명을 작성할 수 있다.
* Architecture Style을 선택할 수 있다.
* Knowledge 설명을 작성할 수 있다.
* Behavior Instruction을 작성할 수 있다.
* 테스트 질문을 통해 Agent 응답 품질을 확인할 수 있다.
* 응답이 부족할 때 Instruction을 수정하여 개선할 수 있다.

---

실습 완료 후 다음과 같은 Agent가 생성됩니다.

| ------------------ | ----------------------------------------------- |
| Agent 이름           | `<자기이름>_AskMkt`                                 |
| Agent 유형           | Marketing Agent                                 |
| Architecture Style | Default                                         |
| 주요 기능              | 마케팅 관련 문의 응답                                    |

| 테스트 방식             | 채팅 패널에서 질문 입력 후 응답 확인                           |

# Step 1. Agent 생성하기

## 1-1. Agent Builder 진입

watsonx Orchestrate 화면에서 Agent를 생성할 수 있는 메뉴로 이동합니다.

일반적인 흐름은 다음과 같습니다.

```text
Build 또는 Agent Builder 메뉴 진입
→ Create Agent 또는 New Agent 선택
→ 처음부터 만들기 선택
```

---

## 1-2. Agent 이름 입력

Agent 이름에 아래 형식으로 입력합니다.

```text
<자기이름>_AskMkt
```

예시:

```text
Seongman_AskMkt
```

---

## 1-3. Agent Description 입력

Description에는 Agent가 어떤 역할을 수행하는지 작성합니다.

아래 내용을 복사하여 Description 필드에 입력합니다.

```text
당신은 사내 직원들의 마케팅 관련 문의를 지원하는 마케팅 업무 지원 에이전트입니다.

주요 역할은 온라인 홍보 마케팅, 키워드 활용, 블로그 마케팅, 카카오스토리 활용, 페이스북 마케팅, 포털사이트 등록 방법에 대한 질문에 답변하는 것입니다.

사용자가 캠페인 기획, 마케팅 자료 요청, 브랜드 가이드라인 확인, 성과 리포트 작성, 소셜 미디어 일정 관리, 이벤트 협업, 예산 추적과 관련된 질문을 하면 명확하고 실무적인 답변을 제공합니다.

답변은 가능한 한 지식 베이스의 내용을 우선 참고하며, 필요한 경우 단계별 설명, 예시, 체크리스트 형태로 정리합니다.
```

---

## Description 작성 시 주의사항

Description은 단순한 소개 문구가 아닙니다.

Agent가 다음을 이해할 수 있도록 작성해야 합니다.

* 나는 어떤 역할을 하는 Agent인가?
* 어떤 업무 질문에 답해야 하는가?
* 어떤 범위의 질문을 처리해야 하는가?
* 어떤 방식으로 답변해야 하는가?

좋지 않은 예시는 다음과 같습니다.

```text
마케팅을 도와주는 에이전트입니다.
```

이 설명은 너무 짧고 역할 범위가 불명확합니다.

좋은 예시는 다음과 같습니다.

```text
온라인 홍보, 키워드 활용, 블로그 마케팅, SNS 채널 운영, 포털사이트 등록 방법에 대한 직원 문의를 처리하는 마케팅 업무 지원 에이전트입니다.
```

---

# Step 2. Architecture Style 선택하기

## 2-1. Architecture Style 확인

Agent 설정 화면에서 Architecture Style을 선택합니다.

이번 실습에서는 다음 값을 선택합니다.

```text
Default
```

---

## 2-2. 왜 Default를 선택하는가?

이번 실습의 목적은 복잡한 도구 실행이 아니라, Knowledge와 Instruction을 기반으로 마케팅 질문에 답변하는 Agent를 만드는 것입니다.

따라서 복잡한 추론이나 계획 실행보다, 질문에 따라 유연하게 답변할 수 있는 **Default Style**이 적합합니다.

---

## 2-3. Architecture Style 비교

| Style    | 특징                     | 적합한 경우                    |
| -------- | ---------------------- | ------------------------- |
| Default  | 프롬프트 중심으로 유연하게 답변      | 단순 질의응답, 정보 안내, 보고서 조회    |
| ReAct    | 생각 → 행동 → 관찰 구조로 반복 수행 | 복잡한 추론, 탐색, 중간 검증이 필요한 작업 |
| Plan-Act | 먼저 계획을 만들고 단계별 실행      | 다단계 워크플로, 실행 순서가 중요한 업무   |

이번 실습에서는 **Default**를 선택합니다.

---

# Step 3. Knowledge 구성하기

## 3-1. Knowledge 섹션으로 이동

Agent 설정 화면에서 **Knowledge** 섹션으로 이동합니다.

Knowledge는 Agent가 참고할 지식의 범위를 설정하는 영역입니다.

---

## 3-2. Knowledge 설명 입력

아래 내용을 복사하여 Knowledge 설명 필드에 입력합니다.

```text
이 지식 베이스에는 온라인 홍보 마케팅, 키워드 활용 가이드, 블로그 마케팅 활용법, 카카오스토리 운영 방법, 페이스북 마케팅 방법, 포털사이트 등록 방법에 대한 사내 마케팅 가이드가 포함되어 있습니다.

사용자가 온라인 홍보 방법, 키워드 발굴, 블로그 콘텐츠 전략, SNS 채널 운영, 카카오스토리 활용, 페이스북 홍보, 포털사이트 등록 절차를 질문할 때 이 지식 베이스를 우선 참고합니다.

예를 들어 다음과 같은 질문에 답변할 수 있습니다.
- 포털 검색으로 키워드 찾는 방법은 무엇인가요?
- 블로그 마케팅에서 효과적인 키워드 전략은 무엇인가요?
- 카카오스토리는 마케팅에 어떻게 활용하나요?
- 페이스북 페이지 운영 시 주의할 점은 무엇인가요?
- 포털사이트 등록은 어떻게 진행하나요?

키워드: 온라인홍보, 키워드활용, 블로그마케팅, 카카오스토리, 페이스북, 포털사이트등록, SNS마케팅, 콘텐츠마케팅, 캠페인기획
```

---

## 3-3. Knowledge 설명이 중요한 이유

Knowledge에 문서를 넣는 것만으로는 충분하지 않습니다.

Agent가 문서를 더 잘 활용하려면 다음 정보가 필요합니다.

* 이 Knowledge에는 어떤 내용이 있는가?
* 어떤 질문에 이 Knowledge를 사용해야 하는가?
* 사용자가 어떤 키워드로 질문할 가능성이 있는가?
* 어떤 업무 상황에서 참고해야 하는가?

---

## 3-4. 좋은 Knowledge 설명 작성 방법

| 작성 기준    | 설명                           |
| -------- | ---------------------------- |
| 구체적으로 작성 | 문서의 주제와 사용 가능한 질문을 명확히 작성    |
| 쉬운 표현 사용 | 전문 용어보다 사용자가 질문할 법한 표현 사용    |
| 키워드 포함   | 검색될 가능성이 높은 단어 포함            |
| 예시 질문 포함 | Agent가 어떤 질문에 답해야 하는지 명확히 안내 |
| 주기적 업데이트 | 문서나 업무 범위가 바뀌면 설명도 수정        |

---

# Step 4. Behavior Instruction 작성하기

## 4-1. Behavior 섹션으로 이동

Agent 설정 화면에서 **Behavior** 섹션으로 이동합니다.

Behavior는 Agent가 사용자의 요청에 어떻게 반응해야 하는지를 정의하는 영역입니다.

---

## 4-2. Instructions 입력

아래 내용을 복사하여 Instructions 필드에 입력합니다.

```text
당신은 사내 마케팅 업무를 지원하는 마케팅 전문가 Agent입니다.

답변 원칙:
1. 사용자의 질문에 대해 지식 베이스를 우선 참고하여 답변합니다.
2. 지식 베이스에 근거가 있는 내용은 명확하고 실무적인 표현으로 설명합니다.
3. 답변은 가능한 한 단계별 절차, 체크리스트, 예시 형태로 정리합니다.
4. 사용자의 질문이 모호하면 바로 단정하지 말고 필요한 정보를 먼저 질문합니다.
5. 지식 베이스에 없는 내용은 추측하지 말고, 일반적인 마케팅 관점의 참고 의견임을 밝혀 답변합니다.
6. 답변은 한국어로 작성합니다.
7. 너무 긴 설명이 필요한 경우 먼저 요약을 제공한 뒤 세부 설명을 이어서 제공합니다.

답변 가능한 주요 주제:
- 온라인 홍보 마케팅
- 키워드 활용 가이드
- 블로그 마케팅 활용
- 카카오스토리 활용법
- 페이스북 마케팅
- 포털사이트 등록 방법
- 캠페인 기획 초안
- SNS 콘텐츠 아이디어
- 마케팅 자료 요청 대응

답변 형식:
사용자가 방법을 질문하면 다음 형식을 따릅니다.

1. 핵심 요약
2. 실행 방법
3. 주의사항
4. 예시 또는 추천 문구

사용자가 캠페인 기획을 요청하면 다음 형식을 따릅니다.

1. 캠페인 목표
2. 타깃 고객
3. 핵심 메시지
4. 활용 채널
5. 콘텐츠 아이디어
6. 실행 체크리스트

사용자가 지식 베이스 범위를 벗어난 질문을 하면 다음과 같이 답변합니다.

"현재 지식 베이스에는 해당 내용이 충분히 포함되어 있지 않습니다. 다만 일반적인 마케팅 관점에서는 다음과 같이 접근할 수 있습니다."
```

---

## 4-3. Behavior 작성 시 핵심

Behavior는 Agent의 답변 품질을 결정합니다.

Description이 Agent의 역할을 정한다면, Behavior는 Agent의 답변 방식을 정합니다.

| 항목          | 역할                    |
| ----------- | --------------------- |
| Description | Agent가 어떤 역할인지 정의     |
| Knowledge   | Agent가 어떤 지식을 참고할지 정의 |
| Behavior    | Agent가 어떻게 답변할지 정의    |

---

## 4-4. 좋지 않은 Behavior 예시

```text
지식 베이스를 참고해서 마케팅 전문가처럼 대답합니다.
```

이 지침은 너무 짧습니다.

다음 항목이 부족합니다.

* 답변 형식
* 모호한 질문 처리 방법
* 지식 베이스에 없는 내용 처리 방법
* 캠페인 기획 요청 시 출력 구조
* 답변 언어
* 주의사항

---

# Step 5. Agent 테스트하기

Agent 생성과 설정이 완료되면 채팅 패널에서 테스트합니다.

아래 질문을 하나씩 입력하고 응답을 확인합니다.

---

## 테스트 1. 키워드 활용 질문

```text
포털 검색으로 키워드 찾는 방법은 무엇인가요?
```

확인할 내용:

* 키워드 조사 방법을 단계별로 설명하는가?
* 포털 검색, 자동완성, 연관 검색어 등을 언급하는가?
* 실무자가 따라 할 수 있는 방식으로 답변하는가?

---

## 테스트 2. 카카오스토리 활용 질문

```text
카카오스토리를 어떻게 활용하나요?
```

확인할 내용:

* 카카오스토리의 활용 목적을 설명하는가?
* 콘텐츠 운영 방법을 제안하는가?
* 마케팅 채널로 활용할 때의 주의사항을 포함하는가?

---

## 테스트 3. 블로그 마케팅 질문

```text
블로그 마케팅에서 효과적인 키워드 전략은 무엇인가요?
```

확인할 내용:

* 키워드 선정 기준을 설명하는가?
* 검색 의도, 콘텐츠 주제, 경쟁도를 연결해서 설명하는가?
* 예시를 포함하는가?

---

## 테스트 4. 캠페인 초안 생성

```text
신제품 출시를 위한 SNS 캠페인 초안을 만들어주세요. 제품은 친환경 텀블러입니다.
```

확인할 내용:

* 캠페인 목표를 제시하는가?
* 타깃 고객을 정의하는가?
* 핵심 메시지를 작성하는가?
* 활용 채널과 콘텐츠 아이디어를 제안하는가?
* 실행 체크리스트를 포함하는가?

---

## 테스트 5. 모호한 질문 처리

```text
이번에 홍보 좀 하려고 하는데 어떻게 하면 될까요?
```

확인할 내용:

* 바로 단정적으로 답변하지 않는가?
* 제품, 타깃, 예산, 기간, 채널 등 추가 정보를 질문하는가?
* 또는 기본적인 접근 방법을 먼저 안내하는가?

---

# Step 6. 응답 품질 개선하기

테스트 결과가 만족스럽지 않다면 Behavior Instruction을 수정합니다.

Agent 개발은 한 번에 끝나는 작업이 아닙니다.

다음 순서로 개선합니다.

```text
질문 입력
→ 응답 확인
→ 부족한 점 확인
→ Behavior 수정
→ 다시 테스트
```

---

## 6-1. 답변이 너무 일반적일 때

문제:

```text
답변이 누구나 할 수 있는 일반적인 설명처럼 보입니다.
```

개선 방법:

Behavior에 아래 지침을 추가합니다.

```text
답변에는 실무자가 바로 적용할 수 있는 구체적인 실행 방법을 포함합니다.
```

또는

```text
답변 마지막에는 항상 "실무 체크포인트"를 3개 이내로 추가합니다.
```

---

## 6-2. 캠페인 초안이 약할 때

문제:

```text
캠페인 기획 요청에 대한 답변이 너무 짧습니다.
```

개선 방법:

Behavior에 아래 지침을 추가합니다.

```text
캠페인 기획 요청에는 반드시 타깃 고객, 핵심 메시지, 활용 채널, 콘텐츠 아이디어, 실행 체크리스트를 포함합니다.
```

---

## 6-3. 모호한 질문에 바로 답변할 때

문제:

```text
사용자가 정보를 충분히 주지 않았는데 Agent가 바로 답변합니다.
```

개선 방법:

Behavior에 아래 지침을 추가합니다.

```text
사용자의 요청이 모호한 경우 제품, 타깃 고객, 예산, 기간, 활용 채널을 먼저 확인합니다.
```

---

## 6-4. 지식 베이스에 없는 내용도 아는 척할 때

문제:

```text
Knowledge에 없는 내용도 확정적으로 답변합니다.
```

개선 방법:

Behavior에 아래 지침을 추가합니다.

```text
지식 베이스에서 확인할 수 없는 내용은 확정적으로 말하지 말고, 일반적인 마케팅 관점의 참고 의견임을 명확히 밝힙니다.
```

# 실습 정리

이번 실습에서는 Marketing Agent를 만들면서 Agent 품질을 결정하는 핵심 요소를 확인했습니다.

Agent를 잘 만들기 위해서는 단순히 LLM을 연결하는 것만으로는 부족합니다.

중요한 것은 다음 세 가지입니다.

1. **역할을 명확히 정의하기**
2. **어떤 지식을 사용할지 설명하기**
3. **어떤 방식으로 답변할지 지침을 주기**

즉, 업무용 Agent의 품질은 다음 흐름으로 개선됩니다.

```text
역할 정의
→ 지식 연결
→ 행동 지침 작성
→ 테스트
→ 지침 개선
```

Marketing Agent 실습은 단순한 챗봇 만들기가 아니라, 실제 기업에서 반복적인 마케팅 문의를 자동화하는 업무 지원 Agent 설계의 기초입니다.

# 마무리 메시지

이번 시간에 만든 Marketing Agent는 다음과 같은 업무에 활용할 수 있습니다.

* 반복적인 마케팅 문의 응대
* 마케팅 가이드 문서 기반 질의응답
* SNS 캠페인 초안 작성
* 키워드 전략 안내
* 블로그 및 SNS 채널 운영 가이드
* 포털사이트 등록 절차 안내

다음 단계에서는 이 Agent에 Tool이나 외부 업무 시스템을 연결하여 더 복잡한 업무 자동화 Agent로 확장할 수 있습니다.
