# 🧑‍💼 4교시 AskHR: API Tool을 활용한 HR 에이전트 만들기

## 1. 실습 소개

이번 실습에서는 IBM watsonx Orchestrate에서 **HR 업무를 지원하는 AskHR 에이전트**를 만듭니다.

AskHR 에이전트는 직원의 HR 관련 질문에 답변하고, 필요한 경우 외부 HR API를 Tool로 호출하여 개인 HR 업무를 처리합니다.

이번 실습에서 에이전트는 두 가지 방식으로 동작합니다.

| 요청 유형       | 처리 방식             | 예시                 |
| ----------- | ----------------- | ------------------ |
| 일반 HR 정책 문의 | Knowledge Base 사용 | 반려동물 정책이 어떻게 되나요?  |
| 개인 HR 업무 요청 | HR API Tool 호출    | 내 프로필 보여줘, 휴가 신청할게 |

핵심 개념은 다음과 같습니다.

```text
정책 질문 → Knowledge Base 사용
개인 HR 업무 → API Tool 사용
```

---

## 2. 실습 목표

이 실습을 완료하면 다음을 할 수 있습니다.

* watsonx Orchestrate에서 HR 에이전트를 생성할 수 있습니다.
* 문서를 Knowledge Base로 등록할 수 있습니다.
* 외부 API 정의 파일을 Tool로 등록할 수 있습니다.
* Instructions를 설정하여 에이전트의 동작 방식을 제어할 수 있습니다.
* Preview와 AI Chat에서 에이전트를 테스트할 수 있습니다.

## 4. 실습 구성

```text

  ↓
AskHR Agent
  ├─ Knowledge Base

  │
  └─ HR API Tools
```

# 핸즈온 가이드

## 7. watsonx Orchestrate 접속

1. IBM Cloud에 로그인합니다.

2. 왼쪽 상단의 햄버거 메뉴를 클릭합니다.

3. **리소스 목록**으로 이동합니다.

4. AI/Machine Learning 섹션에서 **watsonx Orchestrate** 서비스를 선택합니다.

5. **watsonx Orchestrate 시작** 버튼을 클릭합니다.

6. watsonx Orchestrate 화면에서 왼쪽 메뉴를 엽니다.

7. **Build** 또는 **빌드** 메뉴를 선택합니다.

8. **Agent Builder** 또는 **에이전트 빌더**를 클릭합니다.

> 화면 명칭은 실습 환경이나 버전에 따라 조금 다를 수 있습니다.

---

## 8. HR 에이전트 생성

1. **에이전트 생성 +** 버튼을 클릭합니다.

2. **처음부터 만들기**를 선택합니다.

3. 에이전트 이름을 입력합니다.

### 에이전트 이름 규칙

여러 수강생이 같은 환경을 사용하므로 반드시 아래 규칙을 지켜주세요.

```text
<자기이름>_AskHR
```

예시:

```text
Seongman_AskHR
```

4. 설명에는 아래 내용을 입력합니다.

```text
당신은 직원들의 HR 관련 문의를 처리하는 AI 에이전트입니다.

직원 복리후생, 휴가, 유연근무, 반려동물 정책 등 일반 HR 정책 문의에 답변합니다.

또한 직원 프로필 조회, 휴가 잔여일 확인, 직책 변경, 주소 변경, 휴가 신청과 같은 개인 HR 업무를 지원합니다.

개인 정보 조회나 변경이 필요한 요청은 연결된 HR API Tool을 사용하여 처리합니다.

응답은 간결하고 명확하게 제공합니다.
```

5. **생성** 버튼을 클릭합니다.

---

## 9. 에이전트 스타일 선택

에이전트 스타일 섹션에서 **Default**를 선택합니다.

| 스타일     | 설명                                 |
| ------- | ---------------------------------- |
| Default | 모델의 기본 이해, 계획, Tool 사용 능력을 기반으로 동작 |
| ReAct   | 생각, 행동, 관찰을 반복하며 복잡한 문제를 해결        |

이번 실습에서는 **Default**를 사용합니다.

---

## 10. Knowledge Base 등록

이 단계에서는 AskHR 에이전트가 복리후생 관련 질문에 답변할 수 있도록 문서를 등록합니다.

1. 화면을 아래로 스크롤하여 **지식** 섹션으로 이동합니다.

2. **지식 선택**을 클릭합니다.

3. **파일 업로드**를 선택합니다.

4. **다음**을 클릭합니다.

5. `Employee Benefits.pdf` 파일을 업로드합니다.

6. **다음**을 클릭합니다.

7. Knowledge Base 설명에는 아래 내용을 입력합니다.

```text
이 지식 베이스는 회사의 직원 복리후생과 관련된 내용을 포함합니다.

육아휴직, 반려동물 정책, 유연 근무제, 학자금 대출 상환 지원 등 직원 복지 정책에 대한 질문에 답변할 때 사용합니다.

개인 프로필 조회, 휴가 잔여일 확인, 주소 변경, 직책 변경, 휴가 신청과 같은 개인 HR 업무에는 이 지식 베이스가 아니라 HR API Tool을 사용해야 합니다.
```

8. **저장**을 클릭합니다.

---

## 11. HR API Tool 등록

이 단계에서는 AskHR 에이전트가 HR API를 호출할 수 있도록 Tool을 등록합니다.

1. **도구 세트** 섹션으로 이동합니다.

2. **도구 추가 +** 버튼을 클릭합니다.

3. **Add from file or MCP server**를 선택합니다.

   환경에 따라 메뉴 이름이 다음과 같이 보일 수 있습니다.

   * Add from file or MCP server
   * 파일 또는 MCP 서버에서 추가
   * Import from file
   * 파일에서 가져오기

4. **파일에서 가져오기**를 선택합니다.

5. `hr.yaml` 파일을 업로드합니다.

6. **다음**을 클릭합니다.

7. 가져올 작업 목록이 표시되면 모든 작업을 선택합니다.

8. **완료**를 클릭합니다.

---

## 12. 등록되는 HR API Tool

`hr.yaml` 파일을 등록하면 다음과 같은 HR 업무를 Tool로 사용할 수 있습니다.

| Tool 역할   | 사용 예시          |
| --------- | -------------- |
| 직원 프로필 조회 | 내 프로필 정보를 보여줘  |
| 휴가 잔여일 조회 | 내 남은 휴가 일수 알려줘 |
| 직책 변경     | 내 직책을 변경하고 싶어  |
| 주소 변경     | 내 주소를 변경해줘     |
| 휴가 신청     | 휴가 신청할게        |

---

## 13. Instructions 설정

이 단계에서는 에이전트가 언제 Knowledge Base를 사용하고, 언제 API Tool을 호출해야 하는지 설정합니다.

1. **동작** 또는 **Behavior** 섹션으로 이동합니다.

2. **Instructions** 입력란에 아래 내용을 입력합니다.

```text
당신은 직원들의 HR 업무를 지원하는 AskHR 에이전트입니다.

기본 원칙:
- 직원 복리후생, 휴가 정책, 반려동물 정책, 유연 근무제 등 일반 HR 정책 질문에는 Knowledge Base를 사용하여 답변하세요.
- 사용자 개인 정보 조회, 수정, 신청 업무에는 반드시 HR API Tool을 사용하세요.
- 개인 HR 업무를 Knowledge Base만으로 추측해서 답변하지 마세요.
- Tool 호출 결과를 기반으로 답변하고, Tool 결과에 없는 내용은 임의로 만들어내지 마세요.

사용자 이름 처리:
- 사용자가 처음으로 개인 HR 업무를 요청하면 먼저 직원 이름을 물어보세요.
- 이름을 확인한 후 해당 이름으로 HR API Tool을 호출하세요.
- 같은 대화 세션에서 이미 이름을 확인했다면 다시 묻지 마세요.

개인 HR 업무의 예:
- 프로필 조회
- 휴가 잔여일 확인
- 직책 변경
- 주소 변경
- 휴가 신청

직책 변경:
- 사용자가 직책 변경을 요청하면 먼저 변경할 새 직책을 확인하세요.
- 새 직책이 확인되면 HR API Tool을 사용하여 변경하세요.
- 변경 완료 후 변경된 내용을 간단히 요약하세요.

주소 변경:
- 사용자가 주소 변경을 요청하면 먼저 변경할 새 주소를 확인하세요.
- 새 주소가 확인되면 HR API Tool을 사용하여 변경하세요.
- 변경 완료 후 변경된 주소를 간단히 요약하세요.

휴가 신청:
- 사용자가 휴가 신청을 요청하면 휴가 시작일과 종료일을 확인하세요.
- 날짜는 반드시 YYYY-MM-DD 형식으로 변환하여 Tool에 전달하세요.
- 예: 2025년 5월 22일 → 2025-05-22
- 휴가 신청 결과를 사용자에게 간단히 안내하세요.

응답 방식:
- 응답은 한국어로 작성하세요.
- 응답은 간결하고 명확하게 작성하세요.
- 사용자가 추가로 할 수 있는 다음 행동을 안내하세요.
```

3. 입력한 내용이 저장되었는지 확인합니다.

> 일부 환경에서는 Instructions가 자동 저장됩니다.

# 테스트하기

## 15. Preview에서 테스트

오른쪽 Preview 채팅창에서 아래 순서대로 테스트합니다.

테스트 목적은 다음 두 가지입니다.

1. 정책성 질문은 Knowledge Base로 답변하는지 확인합니다.
2. 개인 HR 업무는 HR API Tool을 호출하는지 확인합니다.

---

## 16. 테스트 1: 복리후생 정책 문의

아래 질문을 입력합니다.

```text
반려동물 정책이 어떻게 되나요?
```

### 기대 결과

에이전트가 `Employee Benefits.pdf` 문서를 기반으로 반려동물 정책을 답변합니다.

이 질문은 개인 정보가 필요하지 않으므로 HR API Tool을 호출하지 않아도 됩니다.

---

## 17. 테스트 2: 프로필 조회

아래 질문을 입력합니다.

```text
내 프로필 정보를 보여줘
```

에이전트가 이름을 물어보면 아래 이름 중 하나를 입력합니다.

```text
장원영
```

또는 다음 이름 중 하나를 사용할 수 있습니다.

```text
카리나
안유진
민지
해린
지민
정국
차은우
```

### 기대 결과

에이전트가 직원 이름을 확인한 뒤 프로필 조회 Tool을 호출합니다.

이후 직원 프로필 정보를 요약해서 보여줍니다.

---

## 18. 테스트 3: 직책 변경

아래 질문을 입력합니다.

```text
내 직책을 변경하고 싶어
```

에이전트가 변경할 직책을 물어보면 아래와 같이 입력합니다.

```text
수석 AI 엔지니어
```

### 기대 결과

에이전트가 직책 변경 Tool을 호출하고 변경 완료 결과를 안내합니다.

---

## 19. 테스트 4: 주소 변경

아래 질문을 입력합니다.

```text
내 주소를 변경해줘
```

에이전트가 새 주소를 물어보면 아래와 같이 입력합니다.

```text
서울특별시 강남구 테헤란로 123
```

### 기대 결과

에이전트가 주소 변경 Tool을 호출하고 변경 완료 결과를 안내합니다.

---

## 20. 테스트 5: 휴가 잔여일 확인

아래 질문을 입력합니다.

```text
내 남은 휴가 일수 알려줘
```

### 기대 결과

에이전트가 휴가 잔여일 조회 Tool을 호출하고 남은 휴가 일수를 안내합니다.

이미 같은 대화에서 이름을 입력했다면 다시 이름을 묻지 않아야 합니다.

---

## 21. 테스트 6: 휴가 신청

아래 질문을 입력합니다.

```text
휴가 신청할게
```

에이전트가 휴가 기간을 물어보면 아래와 같이 입력합니다.

```text
2025년 5월 22일부터 2025년 5월 25일까지
```

### 기대 결과

에이전트가 날짜를 아래 형식으로 변환한 뒤 휴가 신청 Tool을 호출합니다.

```text
2025-05-22 ~ 2025-05-25
```

이후 휴가 신청 결과를 안내합니다.

---

## 22. 테스트 7: 변경 결과 확인

아래 질문을 입력합니다.

```text
내 프로필 다시 보여줘
```

### 기대 결과

에이전트가 프로필 조회 Tool을 다시 호출합니다.

앞서 변경한 직책과 주소가 반영되었는지 확인합니다.

```text
우리 회사 반려동물 정책 알려주고, 내 남은 휴가도 확인해줘

```

## 다른 날짜 표현으로 휴가 신청하기

```text
5월 22일부터 5월 25일까지 휴가 신청하고 싶어
```

또는

```text
2025/05/22부터 2025/05/25까지 휴가 신청해줘
```

### 기대 결과

에이전트가 날짜를 Tool에서 사용할 수 있는 형식으로 변환해야 합니다.

```text
2025-05-22
2025-05-25
```

# 실습 결과 정리

이번 실습에서는 HR 업무를 지원하는 AskHR 에이전트를 만들었습니다.

완성된 에이전트는 다음 업무를 수행할 수 있습니다.

| 기능         | 처리 방식          |
| ---------- | -------------- |
| 복리후생 정책 문의 | Knowledge Base |
| 반려동물 정책 문의 | Knowledge Base |
| 프로필 조회     | HR API Tool    |
| 휴가 잔여일 확인  | HR API Tool    |
| 직책 변경      | HR API Tool    |
| 주소 변경      | HR API Tool    |
| 휴가 신청      | HR API Tool    |

---

## 핵심 정리

AskHR 에이전트는 단순한 FAQ 챗봇이 아닙니다.

문서 기반 질문에는 Knowledge Base를 사용하고, 개인별 업무 처리에는 API Tool을 사용합니다.

```text
Knowledge Base = 문서 기반 답변
API Tool = 외부 시스템 조회 및 업무 처리
```

기업 환경에서 AI Agent를 활용할 때 중요한 점은 에이전트가 단순히 답변하는 것을 넘어 실제 업무 시스템과 연결될 수 있다는 것입니다.

이번 실습의 AskHR 에이전트는 그 구조를 이해하기 위한 기본 예제입니다.

---

# 문제 해결

## 1. Tool이 보이지 않는 경우

다음을 확인합니다.

* `hr.yaml` 파일을 업로드했는지 확인합니다.
* 가져올 작업 목록에서 모든 작업을 선택했는지 확인합니다.
* Tool 등록 후 완료 버튼을 클릭했는지 확인합니다.

---

## 2. 에이전트가 Tool을 호출하지 않는 경우

다음을 확인합니다.

* Instructions에 “개인 HR 업무에는 반드시 HR API Tool을 사용”이라는 지침이 포함되어 있는지 확인합니다.
* 질문이 개인 HR 업무에 해당하는지 확인합니다.
* Preview에서 다시 테스트합니다.

예시 질문:

```text
내 프로필 정보를 보여줘
```

---

## 3. 에이전트가 이름을 계속 다시 물어보는 경우

다음을 확인합니다.

* 같은 대화 세션에서 테스트하고 있는지 확인합니다.
* Instructions에 “같은 대화 세션에서 이미 이름을 확인했다면 다시 묻지 마세요”라는 지침이 포함되어 있는지 확인합니다.

---

## 4. 휴가 신청 날짜가 잘못 처리되는 경우

다음을 확인합니다.

* 날짜를 명확하게 입력합니다.
* Instructions에 날짜를 `YYYY-MM-DD` 형식으로 변환하라는 지침이 포함되어 있는지 확인합니다.

권장 입력:

```text
2025년 5월 22일부터 2025년 5월 25일까지 휴가 신청할게
```

기대 변환:

```text
2025-05-22 ~ 2025-05-25
```

---

## 5. 복리후생 질문에 답변을 잘 못하는 경우

다음을 확인합니다.

* `Employee Benefits.pdf` 파일이 정상적으로 업로드되었는지 확인합니다.
* Knowledge Base 설명에 복리후생, 반려동물 정책, 유연 근무제 등의 키워드가 포함되어 있는지 확인합니다.
* 질문을 조금 더 구체적으로 입력합니다.

예시:

```text
회사 반려동물 정책에 대해 알려줘
```

이 구조는 HR뿐 아니라 IT Helpdesk, 영업 지원, 마케팅 요청 처리, 사내 업무 자동화 등 다양한 업무에 확장할 수 있습니다.

```text
문서로 답할 수 있는 질문은 Knowledge Base로 처리하고,

```

이것이 이번 실습의 핵심입니다.
